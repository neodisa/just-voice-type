# Дизайн: выбор языка из полного набора + смена хоткея из меню

**Дата:** 2026-06-04
**Статус:** утверждён
**Файл:** `voice_type.py` (+ новые `languages.py`, `config.py`)

## Проблема

Сейчас в `voice_type.py`:

- Язык захардкожен в трёх местах ([voice_type.py:611-612](../../../voice_type.py), [:629](../../../voice_type.py)): словари `LANG_FLAGS`/`LANG_NAMES` и фиксированный список `(None, "uk", "en", "ru")`. Нельзя выбрать язык вне этого набора и нельзя пометить «свои» рабочие языки.
- Хоткей задаётся только флагом `--hotkey`, в меню показан статичной строкой `Hotkey: …` ([:661](../../../voice_type.py)). Сменить из меню нельзя — нужен перезапуск с другим флагом.
- Выбор языка/модели живёт только в памяти и сбрасывается при перезапуске.

## Цели

1. **Расхардкодить язык.** Полный набор языков Whisper (~99) доступен для выбора. Пользователь помечает галочкой «рабочие» языки — они выносятся наверх меню.
2. **Смена хоткея из меню** через список пресетов, применяется на лету.
3. **Персистентность** выбранных рабочих языков, активного языка и хоткея между запусками.

## Не-цели (YAGNI)

- Выбор модели **не** персистим (не запрошено), но схема конфига оставляет место добавить позже.
- `voice_type_cli.py` не трогаем.
- Живого захвата клавиши для хоткея нет — выбраны пресеты.
- Флаги/эмодзи для каждого из 99 языков не делаем — только названия (флаги не маппятся на языки однозначно).

## Архитектура

Логику делим на три файла, чтобы не раздувать `voice_type.py` (уже 889 строк) и изолировать тестируемые куски:

- **`languages.py`** — `WHISPER_LANGUAGES: dict[str, str]` (код ISO → английское название, ~99 записей). Чистые данные, без зависимостей. Источник: стандартный список языков токенизатора Whisper.
- **`config.py`** — загрузка/сохранение JSON-конфига:
  - `config_path() -> str` — `~/.config/just-voice-type/config.json` (с учётом `$XDG_CONFIG_HOME`, fallback на `~/.config`).
  - `load() -> dict` — читает файл, валидирует, подставляет дефолты; на битом/отсутствующем файле возвращает дефолты и не падает.
  - `save(cfg: dict) -> None` — атомарная запись (временный файл + `os.replace`), создаёт директорию при необходимости.
  - Дефолты: `{"favorite_languages": ["ru", "uk", "en"], "active_language": None, "hotkey": "right_option"}`.
- **`voice_type.py`** — связывает меню rumps с `config` и `languages`.

## Конфиг-файл

Путь: `~/.config/just-voice-type/config.json`

```json
{
  "favorite_languages": ["ru", "uk", "en"],
  "active_language": null,
  "hotkey": "right_option"
}
```

- `active_language: null` означает Auto (автоопределение).
- Грузится при старте `run_app`. Любое изменение в меню немедленно пишется обратно.
- **Первый запуск:** если файла нет — создаётся из дефолтов, засеянных текущими значениями CLI-флагов (`--lang`, `--hotkey`).
- **Приоритет:** конфиг главнее CLI-флагов, если файл существует. `--lang`/`--hotkey` только засевают файл при первом создании. (Автостарт через launch-agent запускает без флагов, поэтому конфиг — единственный источник правды.)

## Фича 1 — меню языков

Структура подменю `Language`:

```
Language ▶
  🌐 Auto                 ✓ (если активен)
  ───────
  English                 ✓ (если активен)   ← избранные ∪ {активный}, по алфавиту названия
  Russian
  Ukrainian
  ───────
  All languages… ▶
     Afrikaans            ✓ (если в избранном)
     Albanian
     …все 99 по алфавиту…
```

**Поведение:**

- **Верхний блок** = `Auto` + избранные языки + активный (если активный не в избранном — включаем, чтобы он всегда был виден). Клик по пункту → язык становится **активным** (`current_lang["value"]`). Галочка (`state=1`) = активный.
- **«All languages…»** = все ~99 языков по алфавиту. Клик → **toggle статуса «избранное»**. Галочка = язык в избранном. Добавление в избранное → язык появляется в верхнем блоке; удаление → исчезает. Активный язык из вида убрать нельзя (если попытались снять у активного — он остаётся виден сверху).
- Галочка имеет **разный смысл по контексту**: в верхнем блоке = активный (radio, один из), в «All languages…» = членство в избранном (multi-select). Осознанный компромисс: rumps даёт только `state` (галочку), другого визуального маркера нет.
- При любом изменении (активный язык / состав избранного) подменю `Language` перестраивается, конфиг сохраняется.

## Фича 2 — подменю хоткея (пресеты)

Заменяет статичную строку `Hotkey: …` на подменю:

```
Hotkey ▶
  Right Option       ✓ (текущий)
  Left Option
  Fn
  Right Command
  Left Command
  Right Control
  Right Shift
  ───────
  F13
  F14
  …
  F19
```

**Поведение:**

- Каждый пункт маппится на строку-алиас, понятную `parse_hotkey` (`right_option`, `left_option`, `fn`, `right_cmd`, `left_cmd`, `right_ctrl`, `right_shift`, `f13`…`f19`) — все они уже поддержаны в `parse_hotkey` ([voice_type.py:493](../../../voice_type.py)).
- Клик → `hotkey_obj_holder["key"] = parse_hotkey(name)`, обновить галочку, записать в конфиг, нотификация `Voice Type → Hotkey: …`.
- **Применяется на лету:** слушатель `on_press`/`on_release` уже читает ключ из `hotkey_obj_holder` — перезапуск listener не нужен.
- **Edge:** если в момент смены идёт запись (`is_down["v"]` истина) — принудительно сбросить состояние записи (`recorder.stop()` без отправки, `is_down["v"]=False`, `state="idle"`), чтобы не зависнуть на старой клавише.

## Обработка ошибок

- Битый/отсутствующий конфиг → дефолты + пересоздание файла, лог в stderr.
- Неизвестный код языка в `favorite_languages` или `active_language` → отбрасываем невалидные коды (проверка по `WHISPER_LANGUAGES`), `active` невалидный → `null` (Auto).
- Неизвестный/неподдержанный хоткей в конфиге → fallback `right_option`, лог.
- Запись конфига атомарная — краш в середине записи не бьёт файл.

## Тестирование

- **`config.py`** — юнит-тесты без GUI: round-trip `save`→`load`; дефолты при отсутствии файла; fallback при битом JSON; валидация (выброс неизвестных кодов языка / хоткея); атомарность (файл не пустой после записи).
- **`languages.py`** — sanity-тест: словарь непустой, содержит ожидаемые коды (`ru`, `en`, `uk`), значения — строки.
- **Меню rumps** — ручная проверка: запустить app → пометить язык в «All languages…» → появился сверху → выбрать активным → сменить хоткей → перезапустить app → убедиться, что выбор сохранился.
