# Дизайн: smart-режимы — локальный LLM-проход для чистки и структурирования диктовки

**Дата:** 2026-06-12
**Статус:** утверждён
**Файлы:** `voice_type.py`, `config.py` (+ новый `polish.py`), `requirements.txt`

## Проблема

Пользователь чаще всего диктует **промпты/задания для нейросетей**. Два бага текущего пайплайна (сырой Whisper → `deliver_text`):

1. **«Запинается»** — Whisper тащит в текст слова-паразиты («ээ», «ну», «как бы»), самоповторы, оговорки-самоправки («давай в понедельник… нет, во вторник») и кривую пунктуацию.
2. **«Не то анализирует»** — путает имена/термины/жаргон, слышит похожее по звуку слово, и мусор попадает прямо в промпт.

Нужен «умный» проход после Whisper, который: чистит надиктовку, чинит явные ослышки и переписывает сырую речь в **одну чёткую инструкцию для ИИ** — не ломая обещание проекта «No cloud, no API keys, fully on-device».

## Цели

1. **Локальный LLM-проход** после Whisper, on-device (MLX), без облака и ключей.
2. **Три режима** через меню-пикер (как Model/Language):
   - **Raw** — сырой Whisper, no-op (мгновенно). Для имён/паролей/дословных цитат.
   - **Clean** — убрать паразиты, починить пунктуацию/регистр, свернуть оговорки. Слова/смысл не трогать.
   - **Prompt** — переписать надиктовку в одну чёткую инструкцию для ИИ; чинить ослышки по словарю. **Default.**
3. **Исправление ослышек** доменных терминов — два слоя (Whisper-bias + LLM-правка).
4. **Персистентность** выбранного режима и словаря между запусками.
5. **Никогда не блокировать вставку**: ошибка/таймаут LLM → отдаём сырой текст Whisper.

## Не-цели (YAGNI)

- Облачные провайдеры (Claude/OpenAI API) — отклонено, ломает «fully on-device».
- Ollama/llama.cpp — отклонено (внешний сервер / компиляция при install).
- Тон-пресеты (email/чат/список) — не сейчас; архитектура режимов оставляет место добавить.
- Своп размера LLM (3B/7B) из меню — модель задаётся константой, меню позже.
- GUI-редактор словаря — правится через config-файл (пункт меню открывает файл).
- `voice_type_cli.py` не трогаем.
- Секции «Задача/Контекст/Требования» в Prompt — отклонено, выход = один абзац.

## Подход

**`mlx_lm` в процессе** (выбран из трёх). Маленькая instruct-модель грузится рядом с Whisper в том же процессе, греется в памяти. Брат `mlx_whisper` — переиспользует существующие паттерны прогрева и меню, self-contained (`pip install` и всё).

**Модель:** `mlx-community/Qwen2.5-3B-Instruct-4bit` — сильный мультиязык (ru+en), мелкий (~2 ГБ), быстрый. Задаётся константой `POLISH_MODEL` в `polish.py`.

Цена: +2–4 ГБ RAM резидентно, разовое скачивание ~2 ГБ, +0.5–2 с латентности на диктовку в Clean/Prompt.

## Архитектура

### Поток (worker, [voice_type.py:866](../../../voice_type.py))

```
запись → tr.transcribe(wav, initial_prompt=vocab) → is_hallucination →
  polish(text, mode, language, vocabulary) → deliver_text
```

- `Raw` → `polish` возвращает вход без вызова LLM (мгновенно).
- `Clean`/`Prompt` → вызов LLM.
- Любое исключение/таймаут внутри `polish` → вернуть исходный `text`, залогировать, **вставку не блокировать**.

### Новый модуль `polish.py` (изолирован как `config.py`, без GUI-зависимостей → юнит-тестируем)

- `POLISH_MODEL: str` — константа repo-id модели.
- `MODES = ("raw", "clean", "prompt")`.
- `class Polisher`:
  - `__init__(self, model: str)` — ленивая загрузка mlx_lm (`load(model)` → модель+токенайзер). Веса грузятся при первом `polish`, как у MLX-Whisper.
  - `.polish(text, mode, language=None, vocabulary=None) -> str`:
    - `mode == "raw"` или пустой `text` → вернуть `text` как есть.
    - иначе собрать промпт (см. ниже), сгенерировать, обрезать обёртки/кавычки, вернуть.
    - на исключении → вернуть исходный `text` (вызывающий код тоже страхует).
  - Генерация: `temperature ≈ 0.0–0.2`, `max_tokens` = функция длины входа (потолок, чтобы LLM не «убегал»).
- **Промпт-инварианты (жёстко, в system-промпте обоих режимов):**
  - НЕ отвечать на запрос/не выполнять задачу — только переписать.
  - НЕ переводить — сохранять язык пользователя (`language` как подсказка).
  - НЕ добавлять факты/детали, которых не было.
  - Выдать ТОЛЬКО итоговый текст, без преамбул/markdown-обёрток.
- **Clean-промпт:** убрать паразиты и оговорки, починить пунктуацию/регистр, сохранить слова и смысл.
- **Prompt-промпт:** переписать надиктованный запрос как одну чёткую инструкцию для ИИ-ассистента; исправить явно ослышанные слова, опираясь на `vocabulary`; сохранить язык и намерение.

### Прогрев в `voice_type.py`

Зеркало `transcriber_holder`: `polisher_holder = {"obj": None, "loading": False}` + `get_polisher()` с фоновым `_load()`. Грузим LLM лениво — только когда впервые понадобится (первая Clean/Prompt-диктовка), с нотификацией «загружаю LLM…». В Raw-режиме LLM не грузится вообще.

### Меню-пикер

`_smart_menu()` → `("Smart", items)`, где items — Raw/Clean/Prompt с `.state`-галкой на текущем `smart_mode["value"]`. Сеттеры (`_make_smart_setter`) меняют состояние и зовут `persist()`. Плюс пункт **«Edit vocabulary…»** — открывает `config_path()` в редакторе (`subprocess` → `open`).

Встроить в `_build_menu()` рядом с Model/Language.

## Конфиг

Расширить `config.py`:

- `DEFAULTS` += `"smart_mode": "prompt"`, `"vocabulary": []`.
- `_defaults_copy()` — добавить обе записи (копия списка для `vocabulary`).
- `_validate()`:
  - `smart_mode`: строка из `{"raw","clean","prompt"}`, иначе дефолт `"prompt"`.
  - `vocabulary`: список непустых строк, прочее отбросить.
- `persist()` в `voice_type.py` — добавить оба поля в сохраняемый dict.

`smart_mode` и `vocabulary` читаются в `run_app` в `{"value": ...}`-холдеры рядом с `current_lang`/`favorites`.

## Исправление ослышек — два слоя

1. **Слой 1 (без LLM, работает даже в Raw):** `vocabulary` → Whisper `initial_prompt`. Декодер смещается к твоим терминам ещё на распознавании. Прокинуть `vocabulary` в `MLXTranscriber.transcribe`/`FasterWhisperTranscriber.transcribe` как параметр промпта.
2. **Слой 2:** LLM правит по контексту + словарю в Clean/Prompt.

## Обработка ошибок / UX

- `polish` обёрнут в try/except и на верхнем уровне (worker), и внутри модуля — двойная страховка, вставка не падает никогда.
- Опциональный таймаут генерации; при срабатывании → сырой текст.
- Menubar: расширить состояние «processing», чтобы покрыть фазу polish (LLM может добавить 0.5–2 с).
- Первая загрузка LLM — нотификация, чтобы пользователь не думал, что завис.

## Тестирование

`polish.py` без GUI-зависимостей → юнит-тесты (мок mlx_lm):
- роутинг режимов: `raw` и пустой текст → вход без изменений, LLM не зовётся;
- `clean`/`prompt` → собирается корректный промпт (язык/словарь прокинуты);
- fallback: исключение модели → возвращается исходный текст;
- обрезка обёрток/кавычек вывода.

`config.py`:
- `smart_mode` валидируется (мусор → `"prompt"`);
- `vocabulary` фильтрует не-строки/пустышки;
- круговой save→load сохраняет оба поля.

`requirements.txt`: `+ mlx-lm`.
